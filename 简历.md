### 一、AI 结果「比对显示」引擎

围绕上述命题，从不同产品的比对需求中提炼出一套递进的渲染方案，在粒度和交互深度上逐层深入：

**字符级 —— 讯飞智检的 Diff 纠错标注。** 产品基于 CGED 2018 TOP1 中文语法检错技术，核心交互是将 6 大类错误（拼写、语法、标点、数字、量和单位、政治领域）及涉政涉黄违禁词检测结果，精准标注到原文对应字符位。基于 `contentEditable` + `Range API` 实现了字符级定位着色引擎，每种错误类型分配独立颜色通道，构成文本"错误热力地图"。同时实现了类 Word 修订模式的逐条审阅卡片——原文片段 → 修改建议 → 结果预览三段并排，支持逐条接受/忽略/二次编辑。对于 10 万字+长文档，采用虚拟滚动（预估高度 + ResizeObserver 动态修正 + 二分查找定位可视区 `startIndex` + 缓冲 overscan）配合增量 Patch 渲染，首屏高亮标注 < 500ms，通过 `requestIdleCallback` 调度低优先级纠错任务避免抢占主线程。

**段落级 —— 文档翻译的双栏同步比对。** 产品支持 PDF/DOCX/PPT/XLS 等 23 种格式文档在线翻译，需保持原排版格式不变，同时将原文与译文段落一一映射、左右对照。基于 Canvas 逐页渲染各格式文档，提取段落盒模型坐标构建 `position map` 索引；通过 IntersectionObserver 锁定当前可视段落，驱动左右两侧同步高亮。对 PPT/XLS 等结构复杂格式实现树形 Diff 标记文本节点增删改，低置信度译文以透明度编码引导用户优先审校。

**像素级 —— 图片翻译的叠加对比。** 图片经 OCR 识别 → 翻译 → 文字回填后，用户需要对比原图与翻译结果。基于 Canvas 双层渲染架构（原图层 + 译文文字层），通过 `globalCompositeOperation` 控制叠加透明度，`clip` 裁剪实现拖拽滑块左右分屏对比。OCR 识别区域以可交互矩形框叠加在原图上，点击查看该区域原文与翻译结果，支持手动修正后回写翻译图层。

**字段级 —— OCR 训练平台的标注比对。** 详情见下文 OCR 平台部分。

### 二、翻译 SaaS 产品的前端架构抽象

7 条翻译产品线初期的技术方案各自独立、UI 割裂、逻辑重复。通过分析共性与差异，抽象出 **输入适配 → 任务调度 → 比对输出** 三层架构：

- **输入层**：`InputAdaptor` 接口统一多模态接入。大文件场景（视频/音频最大 5 小时/1GB）实现了分片上传 + Web Worker 计算 MD5 Hash + 断点续传，上传成功率 85% → 99%+。
- **处理层**：通用翻译任务调度器管理并发控制、进度轮询、结果回调。实时语音翻译和语音转写通过 WebSocket 实现音频流分片传输与翻译结果流式回推，覆盖 PC/H5/微信小程序三端。
- **输出层**：统一调用比对显示组件库，各产品按需编排。浏览器翻译插件（Chrome Extension）同样基于此架构，利用 MutationObserver 监听动态 DOM 实现增量翻译，处理了 iframe 跨域、Shadow DOM 穿透等注入场景。

### 三、LLab 内部 AI 模型平台

各 AI 团队的模型 Demo 技术栈异构（Python/JS/WebGL 等），需要在一个统一平台中集成展示、在线体验与数据集管理。采用 iframe 微前端方案解耦主应用与模型子应用，通过 `postMessage` 通信协议实现主应用与子应用的状态同步，各团队独立部署、动态接入。搭建了模型在线推理沙箱（输入/输出可视化 + 参数调节）、数据集管理（文本/图像/音频多模态预览与版本管理）、Monaco Editor 代码在线预览。模型接入耗时降低 60%+。

### 四、OCR 规则训练平台

平台服务两类用户：业务人员使用预置模板快速完成标准化表单/证照识别，算法工程师通过自定义模板灵活定义非标场景的识别规则。为此设计了双轨编辑器体系：

- **模板编辑器**：预置模板通过配置化定义识别区域坐标与字段映射，开箱即用。自定义模板提供 Drag & Drop 可视化编辑器，用户可自由拖拽创建识别区域（矩形/表格/列表）、配置字段提取规则和校验逻辑。攻克了坐标系的**三层映射**（Canvas 坐标 ⇔ 物理像素 ⇔ 模板比例尺）、多层级嵌套模板的渲染与事件穿透，实现了模板全生命周期管理（创建/编辑/版本 Diff 回溯/导入导出）。
- **标注引擎**：基于 Canvas 实现标注系统，支持矩形框选、多边形标注、文字标注。标注结果与 OCR 识别结果通过叠加层进行字段级交差比对——识别框（蓝）、标注框（绿）、差异区域（红高亮），配合表格字段逐行对比视图快速定位错误。利用空间索引将标注区域 Hit Test 复杂度从 O(n) 降至 O(log n)，基于 OffscreenCanvas 离屏渲染生成置信度热力图，低置信度区域红色标记指导规则迭代。

平台将 OCR 规则训练从"算法工程师手动编码"转变为可视化编排，训练迭代效率提升 3 倍+。
