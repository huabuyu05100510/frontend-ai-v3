### 一、多模态 AI 结果的「比对显示」引擎

无论是翻译还是纠错，AI 输出都需要与原始内容**逐级对齐、逐层对比**。围绕这一需求设计了四套比对渲染方案，在性能和交互深度上持续递进：

**文档翻译 —— 段落级双栏同步比对。** PDF/DOCX/PPT/XLS 等 23 种格式需保持原排版不变，同时将原文与译文段落一一映射、左右对照。核心方案：基于 Canvas 对每种格式做逐页渲染，提取段落盒模型坐标构建 `position map` 索引；通过 IntersectionObserver 锁定当前可视段落，驱动左右两侧同步高亮对应段落。对 PPT/XLS 等结构复杂的格式，实现了树形 Diff 算法标记文本节点的新增/删除/修改，以不同底色区分变更类型，同时对低置信度译文做透明度编码，引导用户优先审校。

**图片翻译 —— 像素级叠加对比。** 图片经 OCR → 翻译 → 文字回填后，用户需对比原图与翻译图。采用 Canvas 双层渲染架构——底层原图、上层译文文字，通过 `globalCompositeOperation` 控制叠加透明度。实现了拖拽滑块对比组件，利用 `clip` 路径裁剪实现左右分屏，分割线两侧分别展示原图和翻译结果。OCR 识别的每个文字区域以可交互矩形框叠加在原图上，点击即可查看该区域的识别原文与翻译结果，支持手动修正后回写至翻译图层。

**讯飞智检 —— 字符级 Diff 纠错标注。** 基于 CGED 2018 TOP1 的中文语法检错技术，需将 6 大类错误（拼写、语法、标点、数字、量和单位、政治领域）精准标注到原文的对应字符位置。基于 `contentEditable` + `Range API` 实现了字符级定位着色引擎，不同错误类型以独立颜色通道区分，形成文本的"错误热力地图"。同时实现了类 Word 修订模式的逐条审阅对比卡片——原文片段 → 修改建议 → 修改后文本三段并排，支持逐条接受/忽略。涉政涉黄违禁词检测结果按风险等级（高危/中危/低危）分层展示。针对 10 万字+长文档场景，采用虚拟滚动（预估高度 + ResizeObserver 获取真实高度动态修正 + 二分查找定位 `startIndex`）配合增量 Patch 渲染，确保高亮标注首屏渲染 < 500ms、滚动帧率稳定 60fps。还通过 `requestIdleCallback` 实现低优先级纠错任务的空闲调度，避免与用户交互争抢主线程。

### 二、翻译产品矩阵的前端架构抽象

面对 7 条翻译产品线各自独立、UI 割裂、逻辑重复的问题，抽象出 **输入适配 → 任务调度 → 比对输出** 三层通用架构。输入层通过 `InputAdaptor` 接口统一多模态接入（文件/文本/语音/图片/视频），大文件场景（视频/音频最大 5 小时/1GB）实现了分片上传 + Web Worker 计算 MD5 Hash + 断点续传，上传成功率从 85% 提升至 99%+。处理层封装了通用翻译任务调度器，管理并发控制、进度轮询和结果回调，实时语音翻译通过 WebSocket 实现音频流分片传输与翻译结果流式回推。输出层统一调用上述比对显示组件库，各产品按需编排组合。同样基于这套架构思想完成了 Chrome Extension 网页翻译插件，利用 MutationObserver 监听动态 DOM 实现增量翻译，处理了 iframe 跨域、Shadow DOM 穿透等注入难题。

### 三、AI 模型平台 & OCR 规则训练平台

**LLab AI 模型平台（类 ModelScope/魔搭社区）**：主导从 0 到 1 的前端架构设计。核心挑战在于各 AI 团队的模型 Demo 技术栈异构、独立迭代需求强烈。采用 qiankun 微前端方案解耦主应用与模型子应用，实现独立部署、动态加载。搭建了模型 Studio（在线推理沙箱 + 输入/输出结果可视化）、数据集管理（文本/图像/音频多模态预览与版本管理）、Monaco Editor 代码在线预览。模型接入成本降低 60%+。

**OCR 规则训练平台**：投入最深、技术密度最高的项目。平台同时服务两类用户——业务人员使用**预置模板**快速完成标准化识别，算法工程师通过**自定义模板**灵活定义复杂规则。为此设计了双轨编辑器体系：

- **模板体系**：预置模板面向固定表单/证照等标准场景，通过配置化方式定义识别区域坐标与字段映射，开箱即用；自定义模板面向非标场景，设计了一套基于 Drag & Drop 的可视化编辑器，用户可自由拖拽创建识别区域（矩形/表格/列表）、配置字段提取规则、设置校验逻辑。攻克了缩放坐标系转换（Canvas 坐标 ⇔ 物理像素 ⇔ 模板比例尺的三层映射）、多层级嵌套模板的渲染与事件穿透等交互难点，实现了模板的全生命周期管理（创建/编辑/版本 Diff 回溯/导入导出）。
- **标注引擎**：基于 Canvas 实现图片标注系统。标注结果与 OCR 识别结果通过叠加层进行字段级交差比对——识别框（蓝色）、标注框（绿色）、差异区域（红色高亮），配合表格字段逐行对比视图快速定位识别错误。利用空间索引（Spatial Indexing）优化 Canvas 内标注区域的点击命中检测（Hit Test），将查找复杂度从 O(n) 降至 O(log n)。同时基于 Canvas 离屏渲染（OffscreenCanvas）生成多图片的置信度热力图，低置信度区域以红色热力标记，直观指导规则迭代方向。
- **架构收益**：将深度学习 OCR 训练流程从"算法工程师手动编码配置"转变为"可视化编排"，业务人员无需编码即可完成规则配置与训练发起，训练迭代效率提升 3 倍+。
