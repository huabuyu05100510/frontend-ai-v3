# 极端性能优化实战（课程大纲）

## 1. 课程目标
- 掌握常见列表优化手段：虚拟列表（Virtual List / react-window）、节点池复用、滚动防抖
- 前端多线程编程范式：了解 Web Worker 基础用法，多 Worker 通信，Transferable Objects 传输
- 突破语言性能瓶颈：引入 WebAssembly (Rust/C++) 实现密集型计算，结合 SIMD 指令集加速
- 具备设计高性能前端架构的能力：能设计基于 SharedArrayBuffer 与 Atomics 的多 Worker 协作方案

## 2. 课程大纲

### 模块一：虚拟列表与虚拟滚动
- 不定高虚拟滚动组件设计
- 如何处理图片异步加载导致的高度变化
- DOM 节点复用 vs 节点池
- ResizeObserver 监听真实高度
- 滚动锚定（Scroll Anchoring）
- IntersectionObserver 图片懒加载

### 模块二：千万级数据表格渲染
- DOM vs Canvas 渲染方案
- Canvas 分块渲染（Tile Rendering）
- 数据层与渲染层分离架构
- Web Worker + SharedArrayBuffer 零拷贝架构
- OffscreenCanvas 离屏渲染
- 脏矩形优化（Dirty Rect Optimization）
- 惯性滚动模拟
- Canvas 内部事件系统 / Hit Detection O(1) 点击检测
- 格网指标 GridMetrics 与二分查找定位

### 模块三：极端性能优化业务场景与度量
- 高频交互类：在线协作白板、实时K线图、高帧率拖拽
- 海量数据类：日志分析平台、大表格、虚拟滚动
- 计算密集类：音视频编解码、文件MD5/SHA哈希、3D模型解析

### 性能瓶颈本质：
- 浏览器像素流水线（Pixel Pipeline）：JS > Style > Layout > Paint > Composite
- 强制同步布局（Forced Synchronous Layout）
- V8 GC "Stop The World"
- JavaScript单线程模型限制，Long Tasks（>50ms）

### 性能度量体系：
- PerformanceObserver 监控 Long Tasks
- 自定义 FPS 监控与卡顿（Jank）检测
- 内存泄漏排查（Detached DOM Trees）
- requestIdleCallback 任务调度器
- 自动化卡顿"录制"埋点 SDK

## 5. 实战案例
### 图片压缩（Rust + WASM）
- 用户上传4K图片(20MB+)，前端进行压缩、裁剪和水印
- 传统Canvas方案导致OOM（内存溢出）
- Rust + WASM 方案：image crate 处理，WebAssembly 线性内存直接操作
- 性能提升 30%-50%

### MD5/SHA大文件哈希计算
- 2GB文件哈希，避免主线程卡死
- Web Worker + WASM + Stream 方案
- 使用 Transferable objects 零拷贝传输
- WASM (Rust) 速度 150-300 MB/s，支持增量计算
