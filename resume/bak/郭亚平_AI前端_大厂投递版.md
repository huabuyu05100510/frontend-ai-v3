## 代表项目

### 在哪儿问问 · AI 行中导游 &emsp; 滴滴 LLab &emsp; 2025.06 – 至今

**问题**：微信小程序原生不支持 EventSource，不同基础库 API 差异大，无法直接接入 LLM 流式输出；AI 生成内容结构动态可变，传统模板方案无法适配。

**方案**：
- 三层流式协议层（传输 / 解析 / 消费）+ 背压控制 + 版本嗅探自动降级 + 指数退避重连，StageStreamManager 统一流式会话生命周期，整套模块与业务完全解耦
- GenUI 三层管道（chunk 解析 → 结构识别 → 组件映射），AI 输出格式与渲染逻辑解耦，新 UI 类型零改动渲染核心
- useChatScroll Hook 解决高频 chunk 更新与用户手势竞争，程序滚动/手势隔离 + 近底阈值自动跟随 + 上划即停
- AdvancedAudioPlayer 多段音频引擎：N+1 预加载消除段间停顿，±2 段滑动窗口解决长路线内存泄漏，live/playback 双模式统一接口
